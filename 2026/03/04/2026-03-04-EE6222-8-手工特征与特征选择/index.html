EE6222 8 手工特征与特征选择

Deyang Zeng
  2026-03-04 2026-03-04 Created   2026-03-04 23:30:52 2026-03-04 23:30:52 Updated  
  • Theory 
 
  • EE6222 
  • | Feature Generation 
  • | Feature Selection 
 
  • Handcrafted Feature Generation（手工特征构造）：从原始数据里“造出”更有用的新特征
  • Feature Selection（特征选择）：从已有特征里“挑出”最有用的一部分

一个是“创造”，一个是“筛选”。

为什么要做特征工程

很多机器学习模型，尤其是传统模型（线性回归、逻辑回归、SVM、决策树、KNN 等），模型效果高度依赖输入特征质量。

原始数据往往有这些问题：

  • 信息表达不直接
  • 特征和目标之间关系不是线性的
  • 有冗余、噪声、无关变量
  • 维度太高，容易过拟合
  • 某些重要模式隐藏在组合关系里

所以我们要做两步：

  1. 把原始数据转成更有表达力的特征
  2. 从中选出最有价值的特征

一句话总结就是：
好的特征，能让简单模型也表现得很好。

Handcrafted Feature Generation 是什么

它指的是：利用领域知识、数据理解和数学变换，人为设计新特征，让模型更容易学习。

它不是让模型自己深度学习特征，而是人来“设计输入表示”。

核心目标

手工特征构造通常是为了：

  • 更好地表达原始信息
  • 让复杂关系变得更容易被模型捕捉
  • 提高预测准确率
  • 增强模型可解释性
  • 降低噪声影响

常见的手工特征构造方法

数值特征的变换

（1）缩放 / 标准化

常见方式：

  • Min-Max Scaling
    把特征缩放到 ([0,1])
  • Z-score Standardization
    变成均值 0、标准差 1

为什么做

对以下模型很重要：

  • KNN
  • SVM
  • 逻辑回归
  • 线性回归（尤其带正则化）
  • 神经网络

因为这些模型对“距离”或“参数尺度”敏感。

直觉

如果一个特征范围是 0 到 100000，另一个是 0 到 1，前者会主导模型。

（2）对数变换（Log Transform）

适合右偏（skewed）分布，比如：

  • 收入
  • 房价
  • 浏览量
  • 交易额

作用

  • 缩小极端值影响
  • 让分布更接近正态
  • 让非线性关系更接近线性

例如：

  • 原始：income
  • 新特征：log(income + 1)

（3）幂变换 / 开方

例如：

  • (x^2)
  • (x^3)
  • (\sqrt{x})

用于让模型学习非线性趋势。

比如房价和面积并不总是完全线性，加入面积平方项可能更合适。

多项式特征（Polynomial Features）

如果原始模型是线性的，但真实关系是弯曲的，可以构造：

  • (x)
  • (x^2)
  • (x^3)

以及交互项：

  • (x_1 x_2)

例子

假设预测学生成绩：

  • 学习时间 study_hours
  • 睡眠时间 sleep_hours

除了单独影响，二者可能有联合作用，所以加入：

  • study_hours * sleep_hours

意义

这能让线性模型也拟合一部分非线性关系。

风险

特征数量会爆炸，容易过拟合。

比如 20 个原始特征做二阶多项式后，特征数会大幅增加。

分箱（Binning / Discretization）

把连续变量分成区间。

例如年龄：

  • 0–18
  • 19–30
  • 31–50
  • 50+

作用

  • 处理非线性关系
  • 增强解释性
  • 降低噪声
  • 对某些模型（如朴素贝叶斯、规则模型）更方便

注意

分箱过粗会丢信息；分箱过细会引入噪声。

类别特征编码

原始类别变量（如城市、性别、专业）不能直接给大多数模型，需要编码。

（1）One-Hot Encoding

比如颜色：

  • red → [1,0,0]
  • blue → [0,1,0]
  • green → [0,0,1]

优点

  • 不会引入大小关系
  • 最常用

缺点

  • 类别很多时维度很高（high cardinality）

（2）Ordinal Encoding

适合有顺序的类别：

  • low = 1
  • medium = 2
  • high = 3

注意

如果类别本身无顺序，不要乱用，否则会误导模型。

（3）Frequency / Count Encoding

用类别出现频率代替类别值。

例如：

  • “北京”出现 25%
  • “上海”出现 18%

适合高基数类别。

（4）Target Encoding（课程里有时也会提到）

用类别对应目标均值来编码。

例如在“用户是否购买”任务中：

  • 某城市历史购买率 = 0.62

风险

如果处理不当，容易数据泄漏（data leakage）。

时间特征构造

原始时间戳常常很难直接用，通常拆成：

  • 年
  • 月
  • 日
  • 星期几
  • 小时
  • 是否周末
  • 是否节假日

例子

订单时间 2026-03-04 21:30

可构造成：

  • month = 3
  • day_of_week = Wednesday
  • hour = 21
  • is_weekend = 0

更高级一点

对周期变量（如小时、月份）可做循环编码：

  • sin(2πt/T)
  • cos(2πt/T)

因为 23 点和 0 点其实很接近，直接用数字 23 和 0 会误导模型。

文本特征构造

如果数据是文本，常见手工特征有：

  • 词袋模型（Bag of Words）
  • TF-IDF
  • 文本长度
  • 标点数量
  • 大写字母比例
  • 情感词数量
  • 关键词是否出现

例子

对垃圾邮件分类：

  • 是否包含 “free”
  • 感叹号个数
  • 邮件长度
  • 大写比例

这些都是很典型的 handcrafted features。

领域知识驱动特征

这是手工特征最核心的部分：用业务理解造特征。

例子 1：金融风控

原始特征：

  • 月收入
  • 总负债

构造新特征：

  • 负债收入比 = debt / income

这个特征往往比原始两个特征更有判别力。

例子 2：教育

原始特征：

  • 出勤率
  • 作业提交率

构造：

  • 学习参与度 = 0.5 × 出勤率 + 0.5 × 作业提交率

例子 3：电商

原始特征：

  • 浏览次数
  • 购买次数

构造：

  • 转化率 = purchase / view

这种“比率”“差值”“平均值”“累计值”非常常见。

聚合特征（Aggregation Features）

对于重复记录或序列数据，可以构造聚合信息：

  • 总和
  • 均值
  • 最大值
  • 最小值
  • 标准差
  • 次数
  • 最近一次间隔

例子

一个用户有很多交易记录，可以为每个用户构造：

  • 平均消费金额
  • 最大消费金额
  • 过去 30 天消费次数
  • 最近一次登录距今天数

这些常常比单条记录更有价值。

好的手工特征应具备什么特点

好的特征通常满足：

  • 相关性高：和目标变量有关系
  • 信息量大：能区分不同样本
  • 噪声少：不容易受偶然扰动影响
  • 可泛化：在测试集上仍然有效
  • 可解释：最好能说明为什么有用

Handcrafted Feature Generation 的风险

过拟合

构造太多复杂特征，训练集效果很好，测试集变差。

特别是：

  • 高阶多项式
  • 过多交互项
  • 针对训练集“量身定制”的规则

数据泄漏（Data Leakage）

这是考试里很常见的重点。

什么叫泄漏

你在构造特征时，不小心使用了“未来信息”或“测试信息”。

例子

你要预测学生期末是否及格，却把“期末总成绩”作为特征的一部分，显然泄漏了答案。

更隐蔽的例子

先用全数据（包括测试集）计算均值再标准化，也可能造成轻微泄漏。

正确做法

  • 只在训练集上拟合变换器（如均值、标准差、编码规则）
  • 再用同样规则变换验证集/测试集

维度爆炸

特征过多会导致：

  • 训练慢
  • 存储大
  • 模型不稳定
  • 更容易过拟合

所以这就引出下一部分：Feature Selection。

Feature Selection 是什么

特征选择是：从已有特征中选出最有用的一个子集，去掉无关、冗余或有害特征。

注意它和降维（如 PCA）不完全一样：

  • Feature Selection：保留原始特征中的一部分
  • Dimensionality Reduction（降维）：把原始特征变成新的低维特征组合

为什么要做特征选择

做特征选择有很多好处：

  • 提高模型泛化能力
  • 减少过拟合
  • 降低训练时间
  • 提升可解释性
  • 降低存储和计算成本
  • 去除噪声和冗余信息

一句话：
不是特征越多越好，而是有用的越多越好。

8. 特征选择的三大类方法

这是最核心的知识点。通常分为：

  1. Filter Methods（过滤法）
  2. Wrapper Methods（包裹法）
  3. Embedded Methods（嵌入法）

Filter Methods（过滤法）

思想

先不训练复杂模型，直接用统计指标衡量每个特征的重要性，然后选出一部分。

它是“模型无关”的，速度快。

常见方法

（1）方差过滤（Variance Threshold）

如果一个特征几乎不变，那它区分样本的能力很弱。

例如一个特征 99.9% 都是 1，几乎没有信息量。

优点

  • 简单
  • 快

缺点

  • 只看自身波动，不看和目标的关系

（2）相关系数（Correlation）

衡量数值特征和目标变量的线性关系。

  • 对回归任务常用 Pearson correlation
  • 值越接近 1 或 -1，线性相关越强

用途

  • 选和目标高度相关的特征
  • 删掉彼此高度相关的冗余特征（多重共线性）

注意

相关系数只能捕捉线性关系，非线性关系可能看不出来。

（3）卡方检验（Chi-Square）

常用于分类任务，衡量离散特征与类别之间是否独立。

  • 如果不独立，说明有关系
  • 关系越强，卡方值往往越大

适用于：

  • 类别特征
  • 或非负离散化后的数值特征

（4）互信息（Mutual Information）

衡量特征和目标之间共享的信息量。

优点

  • 能发现非线性关系
  • 比单纯相关系数更灵活

缺点

  • 计算复杂一些
  • 对样本量敏感

Filter 法总结

优点：

  • 快
  • 简单
  • 适合高维数据预筛选

缺点：

  • 往往单独看每个特征，忽略特征之间组合效果
  • 不一定对应最终模型最优

Wrapper Methods（包裹法）

思想

把“特征选择”当作一个搜索问题：
选一组特征 → 训练模型 → 验证效果 → 再调整特征子集

它直接用模型性能来评估特征集合。

常见方法

（1）Forward Selection（前向选择）

从空集开始：

  1. 先试每个单独特征
  2. 选出使模型提升最大的那个
  3. 再在此基础上一个一个加新特征
  4. 直到没有明显提升或达到停止条件

特点

  • 计算量相对比穷举小
  • 适合从少量重要特征开始构建

（2）Backward Elimination（后向剔除）

从所有特征开始：

  1. 每次删掉一个最不重要的特征
  2. 看模型性能变化
  3. 不断删除，直到性能开始明显下降

特点

  • 适合特征数不太大时
  • 起点是全模型

（3）Recursive Feature Elimination（RFE）

递归特征消除：

  1. 训练模型
  2. 根据特征重要性排序
  3. 删除最弱的一部分
  4. 重复进行

这是很经典的方法。

Wrapper 法优缺点

优点：

  • 通常能得到更适合特定模型的特征子集
  • 能考虑特征之间的联合作用

缺点：

  • 计算非常贵
  • 特征多时很慢
  • 容易过拟合验证集（如果反复调）

Embedded Methods（嵌入法）

思想

在模型训练过程中，模型自己顺便完成特征选择。

也就是说：特征选择嵌入在模型学习里面。

常见方法

（1）L1 正则化（Lasso）

L1 正则会鼓励部分权重变成 0。

  • 权重 = 0 的特征可以视为被剔除
  • 常用于线性模型、逻辑回归

直觉

它不仅拟合数据，还惩罚过多特征，让模型更“稀疏”。

（2）树模型特征重要性

决策树、随机森林、梯度提升树会给出特征重要性，比如：

  • 分裂次数
  • 信息增益
  • Gini reduction

重要性低的特征可被删除。

优点

  • 能处理非线性
  • 能自动建模交互关系
  • 对特征缩放不敏感

注意

树模型的重要性有时会偏向：

  • 取值多的特征
  • 连续特征

所以不能盲信。

Embedded 法优缺点

优点：

  • 效率通常比 Wrapper 高
  • 比 Filter 更贴近模型目标
  • 常常效果较好

缺点：

  • 依赖具体模型
  • 换模型后，选出的特征未必还最优

特征选择时要考虑的两个关键问题

无关特征（Irrelevant Features）

和目标几乎没关系。

比如预测考试成绩时，“学号最后一位数字”大多无关。

这类特征应删除。

冗余特征（Redundant Features）

本身和目标有关，但和其他特征重复表达同样信息。

比如：

  • 身高（厘米）
  • 身高（米）

或者：

  • 总分
  • 各科分数之和

这些特征重复度高，会导致模型复杂度增加，甚至引发多重共线性。

10. Feature Selection 和 Feature Extraction 的区别

很多同学容易混。

Feature Selection

从原有特征里挑子集。
比如 100 个特征选 20 个。

Feature Extraction

把原始特征变成新的特征。
比如 PCA 把 100 维压成 10 维新变量。

区别总结

  • Selection：更可解释，因为保留原特征
  • Extraction：压缩能力强，但新特征可能难解释

常见流程：先构造，再选择

实践中常见顺序是：

  1. 数据清洗
  2. 特征编码 / 变换 / 手工构造
  3. 删掉明显无效特征
  4. 用 Filter 初筛
  5. 用 Wrapper 或 Embedded 精选
  6. 交叉验证评估效果

也就是说：

先把有意义的特征造出来，再从里面挑好的。

一个完整例子

假设你做的是“预测学生是否会挂科”。

原始特征有：

  • 出勤率
  • 作业提交率
  • 每周学习时间
  • 睡眠时间
  • 手机使用时长
  • 家庭收入
  • 是否住宿

第一步：手工特征构造

你可以造出：

  • 学习效率 = 学习时间 / 手机使用时长
  • 参与度 = 出勤率 + 作业提交率
  • 睡眠偏差 = |睡眠时间 - 8|
  • 是否高风险 = 出勤率低且作业提交率低

这些比原始特征更贴近“挂科风险”。

第二步：特征选择

然后用：

  • 相关性分析
  • 卡方检验
  • L1 逻辑回归
  • RFE

筛掉：

  • 和结果几乎无关的特征
  • 与别的特征高度重复的特征

结果

模型通常会：

  • 更准
  • 更稳
  • 更容易解释

考试里常考的优缺点总结

Handcrafted Feature Generation 优点

  • 融入领域知识
  • 提高模型表达能力
  • 让简单模型也能拟合复杂关系
  • 可解释性强

Handcrafted Feature Generation 缺点

  • 依赖人工经验
  • 费时间
  • 容易引入偏差
  • 可能造成过拟合和泄漏

Feature Selection 优点

  • 降低维度
  • 减少过拟合
  • 提升训练效率
  • 提高可解释性

Feature Selection 缺点

  • 可能误删有用特征
  • 不同方法结果不一致
  • Wrapper 计算成本高
  • 结果可能依赖具体模型和数据集

14. 常见面试 / 作业问法

下面这些问法本质上都围绕这个主题：

“为什么标准化后模型效果更好？”

因为某些模型对特征尺度敏感，标准化能避免大尺度特征主导训练。

“为什么要加入交互项？”

因为单个特征可能无法表达联合影响，交互项能帮助模型学习组合关系。

“为什么要做特征选择？”

去掉无关和冗余特征，降低过拟合，提高效率和可解释性。

“Filter、Wrapper、Embedded 有什么区别？”

  • Filter：先用统计指标筛，快但粗
  • Wrapper：用模型性能搜子集，准但慢
  • Embedded：训练时自动选择，折中

“L1 为什么能做特征选择？”

因为 L1 会把部分参数压到 0，从而删除对应特征。

一张总表帮助你记忆

主题 本质 作用 典型方法
Handcrafted Feature Generation 造新特征 提高表达能力 log、分箱、交互项、比率、时间拆分、编码
Feature Selection 选好特征 去噪、降维、提效 Filter、Wrapper、Embedded

再细分：

方法 是否依赖模型 优点 缺点
Filter 弱依赖/基本无关 快、简单 忽略特征组合
Wrapper 强依赖 精度常较高 很慢
Embedded 依赖具体模型 效率和效果折中 有模型偏好

最重要的复习结论

你背这几句，基本就抓住核心了：

  1. 特征工程决定传统机器学习模型的上限。
  2. 手工特征构造是把原始数据变成更有信息量的表示。
  3. 特征选择是从已有特征中删掉无关、冗余、噪声特征。
  4. Filter 快、Wrapper 准但慢、Embedded 折中。
  5. 构造特征时要特别小心过拟合和数据泄漏。
