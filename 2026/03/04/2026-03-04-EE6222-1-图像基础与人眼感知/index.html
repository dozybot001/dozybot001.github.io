这一章在学什么

这一章的核心问题其实只有一句话：

现实世界中的光和颜色，怎样被人眼感知，又怎样被计算机表示、分析和处理。

所以整章的逻辑链条是：

现实场景 → 成像 → 人眼感知 → 数字化表示 → 统计分析（直方图） → 图像处理应用

你可以把它理解成：

  1. 图像是怎么来的
  2. 人为什么能看见、怎么看颜色
  3. 计算机怎么存图像
  4. 怎么用统计方法描述图像亮暗分布
  5. 怎么对图像做增强、分析和应用

Image Formation

图像形成的本质

图像的形成，本质上是：

光源发出光 → 光照到物体 → 物体反射/透射光 → 传感器（眼睛/相机）接收光 → 形成图像

所以图像并不是“物体本身”，而是光与物体相互作用后的结果。

成像的三个关键因素

通常可以把成像分成三个组成部分：

（1）Illumination：照明

就是光源提供的光，比如太阳光、灯光、手电筒。

  • 光源强，图像整体更亮
  • 光源弱，图像整体更暗
  • 光源颜色会影响物体的颜色表现

比如白灯下的白纸和黄灯下的白纸，看起来颜色就不一样。

（2）Reflectance：反射率

物体表面对不同波长光的反射能力不同。

  • 白色物体：反射大部分光
  • 黑色物体：吸收大部分光
  • 红色物体：更容易反射红光，吸收其他颜色

所以我们看到的“物体颜色”，不是物体自己发光，而是它选择性反射某些光。

（3）Sensing：传感/接收

接收反射光的设备可以是：

  • 人眼
  • 相机 CCD/CMOS
  • 医学成像传感器
  • 卫星遥感传感器

传感器把接收到的连续光信号，转成可处理的电信号或数字信号。

常用成像模型

数字图像处理中，经常把图像灰度表示成：

f(x, y) = i(x, y) · r(x, y)

其中：

  • f(x, y)：图像在坐标 (x, y) 处的亮度
  • i(x, y)：illumination，照明分量
  • r(x, y)：reflectance，反射分量

这公式是什么意思？

某个位置看到有多亮，取决于：

  • 那里被照得有多亮（光照）
  • 那里的物体有多能反光（反射）

比如：

  • 黑布在强光下也不会太亮（反射率低）
  • 白纸在弱光下也不会特别亮（照明低）

所以图像亮度是这两者共同作用的结果。

成像中的连续与离散

现实世界中的光是连续的，但计算机不能直接处理连续量，所以要经历：

  1. 空间采样（sampling）
  2. 灰度量化（quantization）

这会在后面的“Digital Image Representation”详细讲。

这一部分常考点

你要会回答这些：

  • 图像形成的基本过程是什么？
  • 图像亮度受哪些因素影响？
  • f(x,y)=i(x,y)r(x,y) 中每项的含义是什么？
  • 为什么物体会显示不同颜色？

Human Perception of Image and Color Image

这一部分是“为什么人能看见图像，尤其是颜色”。

人眼的基本结构（考试常识级）

和图像处理关系最密切的是：

  • Cornea / Lens（角膜/晶状体）：聚焦光线
  • Retina（视网膜）：接收光刺激
  • Optic nerve（视神经）：把信号传给大脑

真正与成像感知最相关的是视网膜上的两类感光细胞：

  • Rods（视杆细胞）
  • Cones（视锥细胞）

Rods 和 Cones 的区别

Rods（视杆细胞）

  • 对弱光敏感
  • 主要负责夜间视觉
  • 不擅长分辨颜色
  • 数量多

所以在昏暗环境下，我们更容易看见轮廓，但颜色不明显。

Cones（视锥细胞）

  • 对强光敏感
  • 负责彩色视觉
  • 能分辨细节
  • 数量较少，但功能更精细

所以白天看颜色清楚，主要靠视锥细胞。

为什么人能看见颜色

因为人眼通常有三类视锥细胞，分别对不同波段更敏感，大致对应：

  • 红（R）
  • 绿（G）
  • 蓝（B）

这就是为什么数字图像常用 RGB 表示颜色：它模仿了人眼的三色感知机制。

可见光范围

人眼能感知的电磁波叫可见光，大约波长范围是：

400nm ~ 700nm

  • 波长短一些：偏蓝紫
  • 波长长一些：偏红

超出这个范围，比如：

  • 紫外线（UV）
  • 红外线（IR）

人眼一般看不见，但机器可以感知。

Brightness, Contrast, Adaptation

这几个词特别常见。

Brightness（亮度）

表示图像整体有多亮。

  • 平均灰度高 → 图像偏亮
  • 平均灰度低 → 图像偏暗

Contrast（对比度）

表示亮部和暗部差异有多大。

  • 对比度高：黑白分明，层次明显
  • 对比度低：灰蒙蒙，看起来发雾

Adaptation（视觉适应）

人眼会自动适应环境亮度变化。

例如：

  • 从黑屋走到阳光下，一开始觉得刺眼
  • 过一会儿适应后就正常了

这说明人眼感知不是线性的、固定的，而是动态调整的。

这也是为什么图像处理有时要做增强：让图像更符合人眼观察习惯。

Color Image 的基本原理

彩色图像的本质是：

每个像素不再只有一个亮度值，而是多个颜色通道值。

最常见的是 RGB 三通道：

  • R：红色强度
  • G：绿色强度
  • B：蓝色强度

比如一个像素 (255, 0, 0) 表示纯红色。

加色模型与减色模型

RGB 是加色模型（Additive Color Model）

适用于显示设备，如：

  • 手机屏幕
  • 电脑显示器
  • 电视

原理是：红、绿、蓝光叠加，越加越亮。

  • R + G + B 全强 → 白色
  • 都为 0 → 黑色

CMY/CMYK 是减色模型（Subtractive Color Model）

适用于印刷。

因为油墨是在“吸收”部分光，而不是发光。

这个你只要知道：

  • 屏幕常用 RGB
  • 印刷常用 CMYK

就够了。

这一部分常考点

  • 人眼两种主要感光细胞是什么？作用分别是什么？
  • 为什么 RGB 可以表示颜色？
  • 亮度与对比度的区别是什么？
  • 可见光范围大概是多少？
  • 为什么人眼在不同亮度环境下感知不同？

Digital Image Representation

这部分最重要。它讲的是：计算机如何表示图像。

数字图像是什么

数字图像就是一个二维离散函数：

f(x, y)

其中：

  • x, y 是像素位置（离散坐标）
  • f(x, y) 是该位置的数值（灰度或颜色值）

简单说：

图像 = 一个由很多小格子（像素）组成的矩阵

Pixel（像素）

像素是数字图像的最小单位。

每个像素记录一个数值：

  • 灰度图：一个值
  • 彩色图：多个值（如 RGB 三个值）

例如一个 4×4 图像，就可以看成一个 4 行 4 列的矩阵。

Sampling（采样）

采样是把连续空间离散化。

也就是：把真实场景切成很多小格子。

采样决定什么？

采样决定空间分辨率（spatial resolution）

  • 采样越密（像素越多），图像越清晰
  • 采样越稀（像素越少），图像越模糊、细节丢失

比如：

  • 1920×1080 比 640×480 细节更多

Quantization（量化）

量化是把连续亮度值离散化。

现实亮度是连续的，但计算机只能记录有限级别。

例如灰度图常用 8-bit：

  • 可表示 2^8 = 256 个灰度级
  • 范围通常是 0 ~ 255

其中：

  • 0 = 黑
  • 255 = 白
  • 中间 = 不同深浅的灰

量化决定什么？

量化决定灰度分辨率（gray-level resolution）

  • 量化级数越多，过渡越平滑
  • 量化级数越少，容易出现“色带”或层次断裂

Spatial Resolution vs Gray-level Resolution

这是最容易混淆的点。

空间分辨率（Spatial Resolution）

描述图像中有多少像素，即细节的空间精细程度。

  • 高空间分辨率：边缘更清晰、细节更多
  • 低空间分辨率：块状感明显

灰度分辨率（Gray-level Resolution）

描述图像中每个像素能表示多少亮度层次

  • 高灰度分辨率：亮度变化更自然
  • 低灰度分辨率：容易出现断层

记忆方法：

  • 采样管“多少个像素”
  • 量化管“每个像素有多少级”

Binary, Grayscale, Color Images

（1）二值图 Binary Image

每个像素只有两种取值：

  • 0（黑）
  • 1（白）

常用于：

  • 文字识别
  • 目标分割
  • 掩膜处理

（2）灰度图 Grayscale Image

每个像素只有一个亮度值。

常见 8-bit 灰度图：0~255。

适合：

  • 边缘检测
  • 纹理分析
  • 医学图像预处理

（3）彩色图 Color Image

每个像素有多个通道值，常见 RGB。

例如 8-bit RGB 图像：

  • 每个通道 0~255
  • 一个像素共 24 bit
  • 可表示约 1677 万种颜色

邻接关系（可作为补充重点）

图像处理中还常讨论像素之间的关系：

  • 4-neighbors（上、下、左、右）
  • 8-neighbors（加上四个对角）

这个在后面学连通性、区域分割时很重要。

这一部分常考点

  • 什么是像素？
  • 采样和量化分别在做什么？
  • 空间分辨率和灰度分辨率有什么区别？
  • 灰度图和二值图、彩色图有什么区别？
  • 为什么 8-bit 灰度图有 256 个灰度级？

最后一个很常考，因为：

2^8 = 256

Image Histogram

直方图是这一章最实用、最高频的内容之一。

什么是图像直方图

图像直方图（Histogram）是：

统计图像中每个灰度值出现次数的分布图。

横轴：

  • 灰度值（如 0~255）

纵轴：

  • 该灰度值出现的像素数量

它不关心这些像素在什么位置，只关心“有多少”。

为什么直方图重要

因为它可以快速反映图像整体亮度分布特征，比如：

  • 图像偏暗还是偏亮
  • 对比度高还是低
  • 灰度分布是否集中
  • 是否适合做增强

所以直方图是图像统计分析的基础工具。

如何理解不同形状的直方图

（1）直方图集中在左边

说明大多数像素灰度低。

→ 图像整体偏暗

（2）直方图集中在右边

说明大多数像素灰度高。

→ 图像整体偏亮

（3）直方图集中在中间且很窄

说明灰度范围小，变化不大。

→ 图像对比度低，发灰

（4）直方图分布很宽

说明灰度覆盖范围大。

→ 图像对比度高，层次丰富

数学表示（考试可能简答）

设灰度级为 r_k，对应像素数为 n_k，总像素数为 n。

则：

  • 直方图：记录每个 r_k 的 n_k
  • 归一化直方图：p(r_k) = n_k / n

其中：

p(r_k) 表示灰度 r_k 出现的概率。

所以归一化直方图可以理解为：

灰度分布的概率分布

Histogram 与图像增强

直方图常用于图像增强，最典型的是：

（1）Contrast Stretching（对比度拉伸）

把原本集中在一个窄范围内的灰度，拉伸到更宽范围。

效果：

  • 暗的更暗一些
  • 亮的更亮一些
  • 对比度提升

（2）Histogram Equalization（直方图均衡化）

目标是让灰度分布更均匀，增强整体对比度。

适合：

  • 细节不明显
  • 灰度集中
  • 整体发灰的图像

但要注意：

  • 它不一定让图像“更自然”
  • 它是增强对比度，不是万能美化

彩色图像的直方图

彩色图像可以：

  • 分别统计 R、G、B 三个通道的直方图
  • 或先转成灰度图再统计

如果直接对 RGB 分别处理，有时会引起颜色失真，所以实际中常要更谨慎。

这一部分常考点

  • 什么是图像直方图？
  • 直方图能反映哪些图像特征？
  • 偏暗/偏亮图像的直方图有什么特点？
  • 什么是归一化直方图？
  • 直方图均衡化的作用是什么？

Image Processing and Its Application

这一部分是总览：图像处理到底是什么，能干什么。

什么是图像处理

图像处理（Image Processing）就是：

对图像进行操作，使其更适合人观察，或更适合机器分析。

它的输入通常是图像，输出可以是：

  • 改进后的图像
  • 提取出的特征
  • 识别结果
  • 决策信息

图像处理的两大目标

（1）Improving pictorial information for human interpretation

为了让人更容易看懂。

例如：

  • 调亮太暗的照片
  • 去噪
  • 锐化模糊边缘
  • 提高对比度

这类偏“增强”。

（2）Processing for autonomous machine perception

为了让机器更容易理解。

例如：

  • 人脸识别
  • 车牌识别
  • 缺陷检测
  • 医学图像诊断辅助

这类偏“分析与识别”。

图像处理的典型步骤

虽然不同任务不同，但通常流程类似：

  1. Image acquisition：图像采集
  2. Preprocessing：预处理（去噪、增强）
  3. Segmentation：分割（把目标和背景分开）
  4. Feature extraction：特征提取
  5. Recognition / Classification：识别分类
  6. Interpretation / Decision：理解与决策

你可以把它想成：

先把图像拍下来 → 调整清楚 → 找出重要区域 → 提取特征 → 判断是什么

常见图像处理操作

（1）Enhancement（增强）

让图像更清晰、更易观察。

  • 调亮度
  • 调对比度
  • 锐化
  • 直方图均衡化

（2）Restoration（复原）

针对已知退化模型去恢复图像。

  • 去模糊
  • 去运动模糊
  • 去噪

它比增强更强调“根据模型恢复真实图像”。

（3）Compression（压缩）

减少图像存储空间与传输成本。

  • JPEG
  • PNG（无损压缩）
  • 医学或工业图像中的专用压缩

（4）Segmentation（分割）

把图像切分成有意义区域。

比如：

  • 从CT图像中分离肿瘤区域
  • 从道路图像中分出车道线
  • 从背景中分离人脸

（5）Recognition（识别）

识别图像中的对象。

  • OCR 文字识别
  • 人脸识别
  • 指纹识别
  • 自动驾驶目标识别

图像处理 vs 计算机视觉

很多老师会顺带讲这个区别。

  • Image Processing：更关注“处理图像本身”
  • Computer Vision：更关注“理解图像内容”

例如：

  • 调整亮度、去噪 → 图像处理
  • 识别这是猫还是狗 → 计算机视觉

你可以理解成：

图像处理常是视觉系统的前置步骤。

把整章串起来记忆

你可以用这条线来背：

（1）图像怎么来

光照到物体，物体反射，传感器接收，形成图像。
图像亮度常建模为：f(x,y)=i(x,y)r(x,y)。

（2）人怎么感知

人眼通过视杆和视锥感知亮度和颜色。
颜色感知与三色机制有关，所以 RGB 很重要。

（3）计算机怎么存

图像是像素矩阵。
通过采样把空间离散化，通过量化把灰度离散化。

（4）怎么分析图像整体特征

用直方图统计灰度分布，判断亮暗与对比度。

（5）怎么利用这些知识

进行增强、复原、压缩、分割、识别，并应用到医学、工业、安防等领域。

你最容易丢分的几个点

采样和量化不要混

  • 采样：决定“像素数量”
  • 量化：决定“灰度级数量”

亮度和对比度不要混

  • 亮度：整体明暗
  • 对比度：亮暗差异程度

图像处理和图像识别不要混

  • 处理：改善图像或提取信息
  • 识别：判断图像里是什么

直方图不反映位置

直方图只统计数量，不告诉你像素在图像中的哪儿。

这章主要介绍数字图像处理的基础概念。首先，图像形成依赖于光源照明、物体反射和传感器接收，图像亮度常表示为照明分量与反射分量的乘积。其次，人眼通过视杆细胞和视锥细胞感知图像，其中视锥细胞使人能够分辨颜色，因此彩色图像通常采用 RGB 模型表示。然后，数字图像可看作由像素组成的二维离散函数，其形成依赖采样和量化，分别决定空间分辨率和灰度分辨率。图像直方图用于描述灰度分布，可反映图像的亮度和对比度，并常用于图像增强。最后，图像处理的目标包括改善图像视觉效果和为机器识别提供输入，广泛应用于医学、工业检测、安防监控和遥感等领域。
