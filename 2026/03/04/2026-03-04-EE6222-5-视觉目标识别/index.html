EE6222 5 视觉目标识别

  2026-03-04 2026-03-04 Created   2026-03-05 12:23:38 2026-03-05 12:23:38 Updated  
  • Theory 
 
  • EE6222 
  • | Visual Object Recognition 
  • | Machine Learning 
 

视觉目标识别（Visual Object Recognition） 的核心问题是：

计算机到底是怎么“看懂”一张图里有什么物体的。

它通常会按一个很自然的发展路线来理解：

  1. Matching（匹配）：像不像以前见过的模板？
  2. Classification（分类）：它属于哪一类？
  3. Machine Learning（机器学习）：不手写规则了，让机器自己从数据里学会分类。

这章在解决什么问题

Visual Object Recognition（视觉目标识别） 的任务是：

给一张图片，让计算机判断里面是什么物体，比如猫、狗、汽车、杯子、人脸等。

比如输入一张猫的照片，系统输出：

  • “这是猫”
  • 或者更进一步：“图片左上角有一只猫”

这章主要讲的是最基本的那部分：
如何从图像中判断物体类别。

先建立一个直觉：人是怎么识别物体的？

人识别物体时，会用到很多线索：

  • 形状（圆的、长的、有轮子）
  • 颜色（香蕉常是黄的）
  • 纹理（毛茸茸、光滑）
  • 局部特征（猫耳朵、汽车车灯）
  • 整体结构（脸有两只眼睛一个嘴巴）

计算机也一样，只是它不能“直接懂”，它必须先把图像转换成数字，再基于这些数字做判断。

所以整个流程本质上是：

图像 → 数值表示（特征） → 判断类别

这就是后面“Matching → Classification → Machine Learning”的主线。

第一阶段：Matching（匹配）

这是最直观、最原始的方法。

什么叫 Matching

Matching 就是：

把当前图像和已知模板进行比较，看它最像谁。

有点像“对答案”：

  • 我有一张标准的猫图片模板
  • 来了一张新图片
  • 我把新图片和模板逐像素比较
  • 如果很像，我就说它是猫

这就是最朴素的“模板匹配（template matching）”。

一个简单例子

假设图像是黑白的，大小都是 ：

模板 A（表示数字 1）：

  • 0 1 0
  • 0 1 0
  • 0 1 0

新输入图像：

  • 0 1 0
  • 0 1 0
  • 0 1 0

完全一致，所以它匹配模板 A。

如果输入变成：

  • 0 1 0
  • 0 1 0
  • 1 1 0

虽然很像，但不完全一样。此时可以用“差异数量”衡量相似度。
差异越小，说明越匹配。

Matching 的基本思想

它的核心逻辑是：

  • 先存一些已知物体的“标准样子”
  • 新图像来了后，和这些样子逐一比较
  • 选最相似的那个

数学上常见的比较方式有：

  • 像素差的绝对值之和
  • 欧氏距离（Euclidean distance）
  • 相关性（correlation）

你不用死记公式，记住一句就够：

Matching 本质就是“比较相似度”。

Matching 的优点

它的优点很明显：

  • 直观
  • 容易理解
  • 实现简单
  • 在“物体外观几乎不变”的情况下效果不错

比如：

  • 工厂里固定角度拍摄零件
  • 屏幕上识别固定字体字符
  • 扫描件里识别标准符号

Matching 的缺点（非常重要）

Matching 的问题也很致命，因为现实图像变化太多：

（1）位置变化

同一只猫可能在图像左边、右边、中间。
模板固定在一个位置时，换个位置就不匹配了。

（2）大小变化

同一个物体可能离镜头远近不同。
大猫和小猫像素完全不一样。

（3）旋转变化

杯子旋转一点，像素排列就变了。

（4）光照变化

亮一点、暗一点，整张图像素值都会不同。

（5）姿态变化

猫坐着、趴着、转头时，外观差异很大。

（6）背景干扰

真实图像背景复杂，模板匹配很容易被背景影响。

小结：为什么 Matching 不够

所以 Matching 的问题在于：

它太依赖“长得几乎一模一样”。

而现实世界中，同类物体并不会总是长得一模一样。

于是我们就从“逐个模板硬匹配”，升级到更抽象的：

Classification（分类）

第二阶段：Classification（分类）

Matching 到 Classification 的思想升级

Matching 是在问：

它像哪一个具体模板？

Classification 是在问：

它属于哪一个类别？

这是一种很重要的思维变化：

  • Matching 关心“是否和某个样本一样”
  • Classification 关心“是否属于这一类共同模式”

例如“猫”这个类别里，所有猫都不一样，但它们有共性：

  • 轮廓
  • 耳朵形状
  • 面部结构
  • 毛发纹理

分类的目标不是找“一模一样”，而是找“同类特征”。

分类的本质

分类可以理解为：

在特征空间里，把不同类别分开。

你可以想象：

  • 每张图像都被转换成一个点
  • 猫的图像点大多聚在某一区域
  • 狗的图像点聚在另一区域
  • 分类器要做的，就是画一条边界把它们分开

这条边界叫：

  • decision boundary（决策边界）

什么叫”特征（Feature）”

这是本章最关键的概念之一。

因为计算机不能直接理解“这是一只猫”，它只能先提取特征。

特征就是：

能帮助区分类别的数值信息。

举例：

  • 边缘数量
  • 圆形程度
  • 是否有对称结构
  • 某些局部区域是否有高亮
  • 颜色分布
  • 纹理粗糙度

对于视觉识别来说，图像先变成“特征向量（feature vector）”，再进入分类器。

为什么特征很重要

好的特征应该做到：

  • 同类相近（猫和猫的特征相似）
  • 异类分开（猫和狗的特征差异明显）

如果特征选得不好，分类再高级也没用。

所以早期计算机视觉里，很多精力都花在：

如何手工设计好的特征。

从图像到特征：常见直觉

这一章通常不会一下子深入很复杂的特征工程，但你要理解几个常见方向。

原始像素作为特征

最简单：

  • 把图像所有像素直接排成一个长向量

比如 灰度图：

  • 一共 1024 个像素
  • 就可以变成一个 1024 维向量

优点：

  • 简单直接

缺点：

  • 对位置、亮度、旋转非常敏感
  • 语义弱，难以泛化

所以原始像素通常不够好。

边缘特征

物体的轮廓往往比颜色更稳定。

比如猫和汽车的轮廓结构差很多。
因此可以先提取图像中的边缘：

  • 哪些地方亮度变化快
  • 哪些地方可能是轮廓线

边缘能帮助计算机关注“形状”。

局部特征

有时不是看整张图，而是看局部关键部分：

  • 眼睛
  • 轮子
  • 门把手
  • 角点
  • 小纹理块

因为同一个物体，即使位置变了，局部关键结构仍可能存在。

这比整图模板匹配更灵活。

统计特征

不是看某一个像素，而是看整体统计：

  • 平均亮度
  • 颜色直方图
  • 纹理分布
  • 某种模式出现频率

这样能减少对微小变化的敏感性。

分类器是怎么工作的

提取好特征后，就进入分类阶段。

最简单的理解：打分

假设有几个类别：

  • 猫
  • 狗
  • 车

分类器会对每个类别打一个分：

  • 猫：8.7
  • 狗：2.1
  • 车：0.4

分数最高的类别就是预测结果。

所以你可以把分类器理解为：

一个“打分系统”。

分类器学习的是什么

它学习的是：

  • 哪些特征更像猫
  • 哪些特征更像狗
  • 哪些特征对区分类别最有用

例如：

  • “有尖耳朵轮廓”对猫的分数加分
  • “有轮胎形状”对车的分数加分

本质上，分类器是在学：

特征和类别之间的对应关系。

线性分类器的直觉

早期最常见的分类器是线性的。

它做的事很像：

  • 把每个特征乘一个权重
  • 再加起来
  • 得到一个总分

如果某个特征对识别猫很重要，它的权重就大。
如果某个特征无关，权重就小。

你可以记成一句话：

线性分类器 = “按重要程度给特征加权打分”。

决策边界的直觉

在二维情况下，分类器可能画一条线把两类分开。
在三维情况下，可能是一个平面。
更高维里就是“超平面”。

这就是“线性分类”的直觉来源。

如果数据分布太复杂，线性边界就不够，需要更复杂的方法。

第三阶段：Machine Learning（机器学习）

这部分是整章的重点升级。

为什么要进入机器学习

前面的方法里，人做了很多事情：

  • 人设计模板
  • 人决定比较规则
  • 人手工设计特征
  • 人规定怎么分类

问题是：

现实图像太复杂，人很难写出适用于所有情况的规则。

所以就出现一个更强的想法：

不要手工规定所有规则，改为让机器从数据中自动学习。

这就是机器学习。

2. 机器学习的核心思想

机器学习的核心是：

给机器大量已标注样本，让它自己总结“什么特征对应什么类别”。

例如训练集中有很多图片：

  • 这张是猫
  • 这张是狗
  • 这张是车
  • 这张是杯子

机器通过反复看这些例子，逐渐学会：

  • 猫通常有哪些模式
  • 狗通常有哪些模式
  • 怎样把它们分开

训练集、测试集

这是必须会的基础概念。

训练集（training set）

用来“学习”的数据。
机器在这些数据上调整参数。

测试集（test set）

用来“检验”的数据。
这些数据在训练时不能看见，用来检查模型是否真的学会了，而不是死记硬背。

如果一个模型训练集上全对，但测试集很差，说明它可能：

  • 过拟合（overfitting）

什么叫”学习”

“学习”在这里不是人那种理解，而是：

自动调整模型参数，使预测结果越来越接近正确答案。

比如分类器里有一堆权重：

  • 有的权重大一点
  • 有的小一点

训练过程就是不断改这些权重，让：

  • 猫图片更容易被判成猫
  • 狗图片更容易被判成狗

所以机器学习的本质就是：

从数据中估计参数。

监督学习（Supervised Learning）

视觉分类里最典型的是 监督学习。

意思是：

  • 输入：图片
  • 标签：正确类别（比如“猫”）

机器通过“图片 + 正确答案”来学习映射关系。

这就像老师批改作业：

  • 你答错了 → 调整
  • 你答对了 → 保持或微调

一个完整的视觉识别流程

你可以把整章压缩成下面这条主线，这很适合考试。

输入图像

给定一张图片。

预处理（可选）

比如：

  • 缩放到统一大小
  • 灰度化
  • 去噪声
  • 亮度归一化

目的：减少无关变化。

特征提取

从图像中提取有判别力的信息：

  • 像素
  • 边缘
  • 纹理
  • 局部模式

分类器判断

使用分类器对特征进行打分或决策，输出类别。

学习参数

用训练数据调整模型，让它在未来的图片上也能正确分类。

Matching、Classification、Machine Learning 三者关系

这是这章最容易考“概念比较”的地方。

Matching

  • 核心：和模板比较相似度
  • 依赖：已有模板
  • 优点：简单直观
  • 缺点：不抗变化，泛化差

一句话：

Matching 是“像谁”。

Classification

  • 核心：根据特征判断类别
  • 依赖：特征表示 + 决策边界
  • 优点：比模板更抽象，更能概括一类物体
  • 缺点：如果特征设计不好，效果有限

一句话：

Classification 是“属于哪一类”。

Machine Learning

  • 核心：从数据自动学分类规则
  • 依赖：训练数据
  • 优点：减少手工规则设计，适应复杂场景
  • 缺点：需要数据；数据不足或质量差会影响效果

一句话：

Machine Learning 是“让机器自己学会怎么分类”。

它们的递进关系

最重要的一句总结：

Matching 是最具体的比较，Classification 是更抽象的类别判断，Machine Learning 是自动学习这种判断规则的方法。

所以这章标题里的 “from Matching to Classification to Machine Learning” 表示的是：

视觉识别方法越来越抽象、越来越智能的发展过程。

为什么”泛化（generalization）”是关键

视觉识别里，真正难的不是认出训练时见过的图，而是：

认出没见过但属于同一类的新图。

这就是泛化能力。

例如模型训练时看过很多猫，但测试时来了一只新猫：

  • 毛色不同
  • 姿势不同
  • 背景不同

如果还能识别出来，才说明它真正学到了“猫的本质特征”，而不是只记住了几张图。

所以机器学习追求的不是“记忆”，而是：

从样本中总结规律，并推广到新样本。

常见困难：为什么视觉识别难

你复习时最好记住“难点”，因为老师很喜欢问“为什么这个问题不简单”。

类内差异大（intra-class variation）

同一类内部差异很大。

例如“椅子”：

  • 木椅
  • 办公椅
  • 折叠椅
  • 沙发椅

外观差别大，但都属于椅子。

类间差异小（inter-class similarity）

不同类可能长得很像。

例如：

  • 狼和狗
  • 橘子和柚子
  • 不同型号的汽车

这会让分类边界更难画。

遮挡（occlusion）

物体可能只露出一部分。
例如猫被桌子挡住一半。

背景复杂

真实图像不是纯净白底。
背景里可能有很多干扰物。

光照与视角变化

同一个物体在不同环境下像素变化巨大。

这章背后的一个核心思想：表示比原图更重要

这章虽然看起来在讲分类，但其实背后有个非常关键的思想：

识别的关键不只是“用什么分类器”，更是“如何表示图像”。

也就是：

  • 如果表示（特征）好，简单分类器也能做得不错
  • 如果表示差，再复杂的分类器也难救

这就是经典计算机视觉里常说的：

Representation matters.（表示很重要）

从经典方法到现代方法的过渡

这章标题里虽然只写到 Machine Learning，但它其实也是在为后面的深度学习铺路。

你可以这样理解发展路线：

  1. Template Matching
    直接和模板比

  2. Hand-crafted Features + Classifier
    人工设计特征，再用分类器判断

  3. Machine Learning
    用数据学习分类参数

  4. Deep Learning（后续章节常见）
    连“特征提取”也让机器自动学

所以现代方法更强，是因为它不仅学习“怎么分类”，还学习“怎么看图”。

十四、考试/复习最容易出的题型

解释题

问：什么是 visual object recognition？

答：

视觉目标识别是让计算机从图像中识别物体类别的过程，通常包括图像表示、特征提取和分类决策。

2. 比较题

问：Matching 和 Classification 的区别？

答：

Matching 通过和具体模板比较相似度来识别；Classification 通过特征和决策边界判断样本属于哪一类，更强调类别共性而不是与单个模板完全一致。

理解题

问：为什么需要 machine learning？

答：

因为真实图像变化复杂，手工设计模板和规则难以覆盖所有情况。机器学习可以利用大量标注数据自动学习特征与类别之间的关系，提高泛化能力。

4. 概念题

问：什么是 feature？为什么重要？

答：

Feature 是从图像中提取出来、能帮助区分类别的数值表示。好的特征能让同类样本更接近、异类样本更分开，因此直接影响识别性能。

分析题

问：视觉目标识别为什么困难？

答：

因为存在类内差异大、类间差异小、遮挡、背景复杂、光照变化、视角变化等问题，使得同一物体在不同图像中的外观可能差别很大。

给你一个最简洁的”整章总总结”

如果你要在几句话里讲清这章，可以直接背这个版本：

视觉目标识别的目标是让计算机判断图像中的物体类别。最早的方法是模板匹配（matching），通过将输入图像与已知模板比较来识别物体，但这种方法对位置、尺度、旋转和光照变化非常敏感。随后发展到分类（classification），即先从图像中提取特征，再根据特征将样本划分到某个类别。为了避免人工编写复杂规则，机器学习进一步通过大量标注数据自动学习特征与类别之间的关系，从而提高识别的准确性和泛化能力。其核心思想是：图像需要先被表示成有判别力的特征，然后再进行分类决策。

这章其实就在讲一个朴素但很深刻的转变：

计算机识别物体，不是靠“死记图片”，而是靠把图像转换成可区分的特征，并从数据中学会如何根据这些特征做判断。
